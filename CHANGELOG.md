# Changelog

## 0.2.0 - 2026-07-22

- Harden Git whitespace validation against user-level `core.whitespace` settings, normalize dependency-generated `uv.lock`, and renormalize the index through `.gitattributes`.

- Detect and rebuild virtual environments whose launchers became invalid after moving the repository.
- Add regression coverage for Fish virtual-environment recovery and installer cleanup.

### Added

- Evidence-annotated YAML parameters and a 379-record evidence registry.
- Parameter-level evidence coverage validation.
- Sensitivity-weighted evidence-priority generation.
- Minor substitution and consumption-prevention outcomes.
- Separation of routine verification checks from full identity proofing.
- Separate retained records, centralized records at risk, transient images, biometric scans, and credential assertions.
- Routine and catastrophic breach-exposure terms.
- Separate per-check and per-proofing variable costs.
- Reproducible JSON run manifest with file hashes and runtime versions.
- Quebec Bill 9 legal snapshot and regulation watcher.
- Distributional research registry.
- Stakeholder interview, consent, note, and sampling materials.
- English and French executive summaries.
- GitHub issue templates, CI workflow, and research-issue creation script.
- Four charts and a self-contained static dashboard.
- Native fish Momiji installer, experiment runner, environment check, issue creator, smoke test, and release preflight with no Bash dependency on the fish path.
- An Arch Linux GitHub Actions job for native fish syntax, installer, and end-to-end workflow validation.
- Pull-request-first publishing guide, release notes, and v0.2.0 release checklist.

### Changed

- The primary effectiveness metric now distinguishes transaction prevention from estimated consumption prevention after substitution.
- Expected breach exposure is based on centralized records at risk and retention, not simply annual records created.
- Reusable credentials are modeled through an identity-proofing rate and routine age assertions.
- Costs include routine verification, full identity proofing, and fixed annual costs.

### Evidence warning

The release is a research scaffold and evidence baseline. Most operational inputs remain analyst-selected ranges or explicit research gaps.
